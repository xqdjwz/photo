最简单，零代码，马上就能用的实现方式，不用服务器，不用写程序，客户输入快递单号就能看到你指定的照片。
原理；客户打开一个网页-输入快递单号-系统自动返回你提前设置好的照片。
最终效果；打开网页-输入快递单号-点击查询-显示对应照片

**搭建教程；**
1，新建一个txt文本，复制粘贴代码，然后保存。重新命名为index.html（直接点击在本地电脑运行查看网页效果），
2，注册一个Cloudflare账户，选择左侧Compute，点击Worker&Pages，
3，创建应用程序，项目名称随便填（比如xqdj-photo），其他默认，
4，选择Pages 开始使用，上传项目（找到保存的html文件，拖拽或者选择上传），点击部署，
5，部署完成后，Pages会给你一个xxx.pages.dev网址，直接访问网址就能看到效果了.

**使用教程；**
输入单号：在输入框里填 TEST001（你已经填好了）
点击查询：点绿色的「查询」按钮
查看图片：按钮下方会自动弹出对应图片，显示「这是为您准备的照片」
测试其他单号：清空输入框，输入 TEST002/TEST003，点查询，会显示不同的风景/猫咪图
测试错误单号：输入随便一个不存在的单号（比如 123），点查询，会弹出「单号不存在」的提示

**可能遇到的问题；**
文件后缀错了：Mac 默认会隐藏后缀，你可以右键文件 →「显示简介」，确认「名称与扩展名」里是 index.html，不是 index.html.txt
图片链接失效：图片必须是可直接访问的公网链接（比如 Cloudflare R2、图床），本地文件路径不行
格式不是纯文本：如果代码里出现了字体、颜色、排版，就是没切纯文本，
